## Context

目前「小公主注音練習本」具備基本的怪物進度系統，當打完第 6 關（魔王）後，僅會顯示一個簡單的糖果 Emoji 獎勵。使用者希望增加更具吸引力的貼紙收集機制，提升練習的趣味性。

## Goals / Non-Goals

**Goals:**
- 實作貼紙獎勵系統，擊敗魔王後可獲得一張貼紙。
- 提供「貼紙收集冊」UI，讓使用者查看已收集的成果。
- 使用 `localStorage` 進行持久化儲存。
- 貼紙設計需為可愛童趣風格，且不使用 Emoji（改用 CSS Art 或 Inline SVG）。

**Non-Goals:**
- 本階段不實作貼紙間的等級或稀有度屬性（除外觀區別外）。
- 不實作多個存檔功能。
- 不實作社群分享功能。

## Decisions

### 1. 貼紙資源呈現 (Sticker Assets)
- **決策**：使用 **Inline SVG + CSS Animation**。
- **理由**：
    - 不需要額外的圖片檔案，減少維護成本與載入延遲。
    - 方便實作「灰階（未獲得）」與「彩色（已獲得）」的視覺切換。
    - 在視網膜螢幕上保證絕對清晰。

### 2. 資料儲存 (Storage)
- **決策**：使用 `localStorage` 儲存鍵值 `baby_typing_rewards`。
- **資料結構**：`{ collectedIds: [1, 5, 8], lastAwardedId: 1 }`。

### 3. UI 整合方式
- **決策**：
    - 在左側 `game-status-bar` 底部新增一個「📜 收集冊」圓形按鈕。
    - 使用全螢幕 Modal 顯示收集牆（Sticker Wall）。
    - 獎勵動畫：在 `showCandyReward` 的位置，先播放貼紙抽獎動畫，最後貼紙縮小並「飛向」左側收集冊按鈕。

### 4. 貼紙設計清單 (Sticker Concepts)
1. **Q版奧特曼**：幾何圖形組合的圓潤超人。
2. **皮卡丘**：閃電尾巴與紅臉頰。
3. **胖丁**：粉圓形的歌唱角色。
4. **米奇**：標誌性的三圓頭像。
5. **維尼**：黃色圓臉與紅色衣服。
6. **馬力歐**：紅帽子與大鬍子。
7. **奇諾比奧**：紅白斑點香菇頭。
8. **太空貓**：戴著圓形頭盔的小貓。
9. **注音精靈**：長了翅膀的英文字母。
10. **金龍**：代表最後勝利的金色小龍。

## Risks / Trade-offs

- **[Risk]**：CSS Art 程式碼量較大，會增加 `index.html` 的行數。
    - **Mitigation**：將 SVG 定義封裝在 JavaScript 物件中，動態插入 DOM，保持 HTML 結構整潔。
- **[Risk]**：移動端點擊按鈕可能會有延遲或誤觸。
    - **Mitigation**：使用 `Pointer Events` 並確保按鈕有足夠的 `touch-target` 大小。
